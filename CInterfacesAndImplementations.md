
1. 简介
=======

1.2 编程风格
------------
* 通常，对全局变量和函数使用较长的、容易称呼的名字，而对局部变量使用短的名字。

* 代码中几乎没有什么注释，因为包含代码的代码块周围的上下文代替了注释。


1.3 效率
--------
* 不要过早优化，除非是明显的代码执行效率问题，否则通常重构比优化更为重要。



2. 接口与实现
=============

2.4 客户调用程序的责任
---------------------
* 接口通常说明的是不可检查的运行期错误（unchecked runtime error）、可检查的运行期
  错误（checked runtime error）以及异常（exception）。

* 不可检查运行期错误和可检查运行期错误不是客户调用程序造成的，例如打开文件错误等等。
  异常是一些可能发生但是很少会发生的情况，程序可以从异常中恢复。

* 不可检查的运行期错误不要求实现能够检查出来，可检查的运行期错误是实现保证能够检查
  出来的。典型的如assert()实现了可检查的运行期错误。
